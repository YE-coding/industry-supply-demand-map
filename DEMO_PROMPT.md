# 行业认知图谱 Demo 制作规格

## 新定位

这个网站不是单纯展示 skill，也不是投研报告列表。它的目标是让一个对行业完全不了解的人，快速建立行业的上下游、利益走向、供需矛盾和周期位置。

核心短句：

```text
看懂行业
```

## 体验结构

1. 第一屏白底黑字，只显示“看懂行业”。
2. 用户点击或滑动时，文字像电影字幕一样放大、淡出，节奏要慢一点、优雅一点，不能像硬切。
3. 主界面顶部是类似 ChatGPT 网页版的小搜索框，不占据页面主体。
4. 页面主体是 Obsidian Graph View 风格的行业关系图：
   - 圆点代表行业或关键概念。
   - 线代表产业链或需求、供给、瓶颈关系。
   - 不同分类用莫兰迪/马卡龙式淡色区分，避免高饱和和黑色描边。
   - 悬停节点时节点放大，并高亮相关连接。
   - 屏幕足够大时显示行业简称，默认保持干净。
   - 支持鼠标滚轮缩放和拖拽移动整张图谱。
   - 图谱结构采用“中心分析框架 + 内圈分析维度 + 外圈行业节点”，避免节点互相重叠。
5. 点击某个行业后：
   - 背景图谱和搜索框淡化。
   - 被选中的行业节点移动到左上角并放大。
   - 右侧展开行业详情面板。
6. 详情面板重点展示：
   - 一句话判断。
   - 上下游产业链横向传导轴。
   - 悬停或点击产业链节点时显示节点说明。
   - 关键瓶颈。
   - 利益走向、供需信号和周期位置使用 PPT 式“左文字、右图表”展示。
   - 来源、数据时效、关键量化指标。

## 数据要求

从当前目录的行业供需周期分析 `.md` 报告自动提取：

- 标题
- 分析日期
- 地理范围
- 数据时效
- 一句话判断
- 关键瓶颈
- 产业链代码块
- 来源线索
- 关键量化指标

原始 `.md` 报告只读，不修改。

## 设计参考

参考 Obsidian Graph View 的交互原则：

- 节点代表笔记或对象。
- 线代表对象之间的关系。
- 悬停节点时高亮连接。
- 点击节点打开详情。
- 支持搜索过滤、颜色分组、文本淡显、节点大小和连线厚度。

## 边界

继续保留 skill 的边界：这不是荐股工具，不输出买卖建议、目标价或短线判断。资本市场部分只用于理解产业现实与市场预期之间的关系。
