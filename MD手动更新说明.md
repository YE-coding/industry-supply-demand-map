# 行业 Markdown 手动更新结构

## 更新规则

网站以“标准化后的产业名称”为唯一键：

- 上传的 Markdown 与现有产业同名：覆盖该产业节点；
- 上传的是新产业：创建新节点；
- 其他产业：保持不变；
- 删除本地上传版本：如果存在内置基础版本，则自动恢复基础版本。

标准化时会忽略大小写、空格及产业名末尾的“行业”二字。例如“半导体”和“半导体行业”视为同一产业。

## 操作步骤

1. 打开网站，点击“上传报告”。
2. 选择更新后的 `.md` 文件。
3. 点击“解析并检查”。
4. 网站会提示这是“覆盖现有节点”还是“创建新节点”。
5. 点击确认按钮完成更新。

更新结果保存在当前浏览器的 `localStorage` 中，不会修改磁盘上的原始 Markdown。

## 初始案例更新

网站启动或正式构建时仍会运行 `scripts/extractCases.mjs`，把根目录中的行业报告抽取到 `src/data/cases.json`：

```powershell
npm run dev
npm run build
```

手动上传版本的优先级高于这些内置基础案例。
